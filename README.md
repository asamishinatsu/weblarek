# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

В приложении используются две ключевые сущности для описания данных: **товар** и **покупатель**.

#### Интерфейс товара (IProduct)
Описывает структуру данных отдельного товара (Product).
```typescript
interface IProduct {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number | null;
} 
```
Используется для типизации товаров в каталоге, при детальном просмотре и добавлении в корзину.

#### Интерфейс покупателя (IBuyer)
Описывает данные (Buyer), которые пользователь заполняет при оформлении заказа.
```typescript
type TPayment = 'cash' | 'card';

interface IBuyer {
  payment: TPayment | '';
  address: string;
  email: string;
  phone: string;
} 
```
Используется для типизации и хранения контактных данных и выбранного способа оплаты при передаче заказа на сервер.


### Модели данных

#### 1. Класс `Catalog` (Каталог товаров)
Отвечает за работу с массивом товаров, доступных в магазине, и за хранение состояния товара, выбранного для детального просмотра.

**Конструктор:**
- `constructor(events: EventEmitter)` - принимает брокер событий для инициализации событий при обновлении каталога или выборе товара.

**Поля класса:**
- `items: IProduct[]` - массив всех загруженных товаров.
- `selectedItem: IProduct | null` - товар, выбранный для подробного отображения в модальном окне.
- `events: EventEmitter` - экземпляр брокера событий.

**Методы:**
- `setItems(items: IProduct[]): void` - сохраняет массив товаров и вызывает событие обновления списка.
- `set preview(item: IProduct)` - сохраняет товар для подробного отображения и вызывает событие открытия модального окна.
- `getItems(): IProduct[]` - возвращает массив всех товаров в каталоге.
- `get preview(): IProduct | null` - возвращает выделенный товар.
- `getProduct(id: string): IProduct | undefined` - возвращает товар из каталога по его `id`.

#### 2. Класс `Cart` (Корзина)
Отвечает за управление списком товаров, которые пользователь добавил к покупке.

**Конструктор:**
- `constructor(events: EventEmitter)` - принимает брокер событий для инициализации событий при изменении состава корзины.

**Поля класса:**
- `items: IProduct[]` - массив товаров, добавленных в корзину.
- `events: EventEmitter` - экземпляр брокера событий.

**Методы:**
- `add(item: IProduct): void` - добавление товара в корзину.
- `remove(id: string): void` - удаление товара из корзины по `id`.
- `getCount(): number` - возвращает количество товаров в корзине.
- `getItems(): IProduct[]` - возвращает массив товаров, которые находятся в корзине.
- `getTotal(): number` - вычисляет и возвращает итоговую стоимость всех товаров в корзине.
- `contains(id: string): boolean` - проверка наличия товара в корзине по `id`. Возвращает `true`, если товар уже добавлен.
- `clear(): void` - полная очистка корзины.

#### 3. Класс `Buyer` (Покупатель / Данные заказа)
Отвечает за хранение, управление и валидацию данных пользователя в процессе оформления заказа.

**Конструктор:**
- `constructor(events: EventEmitter)` - принимает брокер событий для инициализации событий при изменении данных или ошибках валидации.

**Поля класса:**
- `payment: TPayment | ''` - вид оплаты (онлайн или при получении).
- `address: string` - адрес доставки.
- `email: string` - контактный email.
- `phone: string` - контактный телефон.
- `events: EventEmitter` - экземпляр брокера событий.

**Методы:**
- `setInfo(field: keyof IBuyer, value: string): void` - обновление конкретного поля по ключу. После записи автоматически вызывает событие обновления.
- `getInfo(): IBuyer` - возвращает все данные покупателя в виде объекта.
- `validateOrder(): ValidationErrors` — проверяет поля первого шага оформления (`payment`, `address`).
- `validateContacts(): ValidationErrors` — проверяет поля второго шага оформления (`email`, `phone`).
- `validate(): ValidationErrors` — проверяет все поля разом. Правило валидации: поле не должно быть пустым. Возвращает объект, в котором ключами выступают названия полей с ошибками, а значениями - текст ошибки. Если все поля валидны, возвращает пустой объект.
*Пример возвращаемого значения при ошибках:*
```javascript
{
  payment: 'Не выбран вид оплаты',
  address: 'Не указан адрес доставки'
}
```
- `clear(): void` - очистка данных покупателя (сброс полей до начальных пустых значений).

### Слой коммуникации

#### Класс `LarekAPI`
Отвечает за взаимодействие с backend-частью приложения.

**Конструктор:**
- `constructor(api: IApi)` - принимает готовый экземпляр класса для работы с сетью.

**Поля класса:**
- `api: IApi` - экземпляр базового класса, через который осуществляются HTTP-запросы (`get`, `post`).

**Методы:**
- `getProductsList(): Promise<IProductList>` - выполняет GET-запрос на эндпоинт `/product/`. Возвращает промис с объектом, содержащим общее количество товаров (`total`) и массив самих товаров (`items`).
- `postOrder(order: IOrder): Promise<IOrderResult>` - выполняет POST-запрос на эндпоинт `/order/`, передавая данные заказа (данные покупателя, сумму заказа и массив ID товаров). Возвращает промис с объектом ответа от сервера, содержащим ID созданного заказа и списанную сумму.

### Классы представления

Каждый класс представления отвечает за отображение своего блока разметки и прослушивание действий пользователя с последующей генерацией событий для презентера. Все классы отнаследованы от базового абстрактного класса `Component`.

#### 1. Общие классы-родители

**Класс `Card`**
Родительский класс, содержащий общий функционал для отображения карточек товара. Не используется напрямую.
- **Конструктор:** `constructor(container: HTMLElement)`
- **Поля:** `titleElement`, `priceElement`, `imageElement`, `categoryElement`, `descriptionElement` — ссылки на DOM-элементы карточки.
- **Методы:**
  - `set id(value: string)` — сохраняет `id` карточки в `dataset`.
  - `getId(): string` — возвращает сохраненный `id`.
  - Сеттеры `title`, `price`, `image`, `category`, `description` — обновляют текстовое содержимое и атрибуты соответствующих DOM-элементов.

**Класс `Form`**
Родительский класс для всех форм в приложении. Обрабатывает общий функционал: вывод ошибок валидации и блокировку кнопки сабмита.
- **Конструктор:** `constructor(container: HTMLElement)`
- **Поля:**
  - `formButtonElement: HTMLButtonElement` — кнопка отправки формы.
  - `formErrorsElement: HTMLElement` — блок для вывода текста ошибок.
- **Методы:**
  - `set valid(value: boolean)` — переключает состояние доступности кнопки (disabled).
  - `set errors(errors: string[])` — устанавливает текст ошибок валидации.

#### 2. Дочерние классы карточек

**Класс `CardCatalog`**
Отвечает за отображение карточки товара на главной странице.
- **Конструктор:** `constructor(container: HTMLElement, events: IEvents)` — дополнительно навешивает обработчик клика на саму карточку, генерируя событие `preview:select`.

**Класс `CardPreview`**
Отвечает за отображение детальной информации о товаре в модальном окне.
- **Конструктор:** `constructor(container: HTMLElement, events: IEvents)` — навешивает обработчик клика на кнопку «В корзину/Удалить из корзины», генерируя событие `cart:toggle`.
- **Поля:** `addToCartButtonElement: HTMLButtonElement`.
- **Методы:** `set buttonState({ inCart, available })` — управляет текстом и активностью кнопки в зависимости от того, есть ли товар в корзине и имеет ли он цену.

**Класс `CardCart`**
Отвечает за компактное отображение товара в списке корзины.
- **Конструктор:** `constructor(container: HTMLElement, events: IEvents)` — навешивает обработчик клика на иконку удаления, генерируя событие `cart:remove`.
- **Поля:** `itemIndexElement: HTMLElement`, `deleteButtonElement: HTMLButtonElement`.
- **Методы:** `set index(val: number)` — задает порядковый номер товара в списке.

#### 3. Дочерние классы форм

**Класс `FormOrder`**
Отображает первый шаг оформления заказа (выбор оплаты и ввод адреса).
- **Конструктор:** `constructor(container: HTMLElement, events: IEvents)` — слушает события клика по кнопкам оплаты (`order:payment`), ввода адреса (`order:address`) и отправки формы (`order:submit`).
- **Поля:** `paymentButtonElements: HTMLButtonElement[]`, `addressElement: HTMLInputElement`.
- **Методы:**
  - `set payment(val: string)` — выделяет активную кнопку способа оплаты.
  - `set address(val: string)` — заполняет поле адреса.

**Класс `FormContacts`**
Отображает второй шаг оформления заказа (ввод Email и телефона).
- **Конструктор:** `constructor(container: HTMLElement, events: IEvents)` — слушает изменения инпутов (`order:email`, `order:phone`) и отправку формы (`order:submit`).
- **Поля:** `emailElement: HTMLInputElement`, `phoneElement: HTMLInputElement`.
- **Методы:** сеттеры `email` и `phone` для обновления значений полей.

#### 4. Глобальные компоненты интерфейса

**Класс `Gallery`**
Отображает сетку каталога товаров на главной странице.
- **Конструктор:** `constructor(container: HTMLElement)`.
- **Методы:** `set catalog(items: HTMLElement[])` — очищает контейнер и добавляет в него переданный массив DOM-элементов карточек.

**Класс `Header`**
Отображает шапку сайта и счетчик товаров на иконке корзины.
- **Конструктор:** `constructor(container: HTMLElement, events: IEvents)` — навешивает событие открытия корзины (`cart:show`).
- **Поля:** `cartButtonElement: HTMLButtonElement`, `counterElement: HTMLElement`.
- **Методы:** `set counter(val: number)` — обновляет числовое значение на счетчике корзины.

**Класс `Cart` (Корзина)**
Отображает список добавленных в корзину товаров и итоговую сумму.
- **Конструктор:** `constructor(container: HTMLElement, events: IEvents)` — обрабатывает клик по кнопке «Оформить» (`checkout:start`).
- **Поля:** `cartListElement: HTMLUListElement`, `cartButtonElement: HTMLButtonElement`, `cartTotalElement: HTMLElement`.
- **Методы:**
  - `set cartItems(items: HTMLElement[])` — выводит список элементов `CardCart`.
  - `set total(val: number)` — выводит общую стоимость.
  - `set isEmpty(value: boolean)` — деактивирует кнопку оформления, если корзина пуста.

**Класс `ModalContainer`**
Базовый самостоятельный компонент модального окна. В него не зашита конкретная разметка, он лишь предоставляет контейнер для отображения любого переданного контента.
- **Конструктор:** `constructor(container: HTMLElement, events: IEvents)` — слушает клик по крестику или вне окна (оверлей) и генерирует событие `modal:close`.
- **Поля:** `closeButtonElement: HTMLButtonElement`, `contentElement: HTMLElement`.
- **Методы:**
  - `set content(content: HTMLElement)` — вставляет переданный DOM-элемент внутрь модального окна.
  - `open() / close()` — управляют CSS-классом активности окна.
  - `clear()` — очищает содержимое контейнера `contentElement`.

**Класс `SuccessModal`**
Отображает уведомление об успешном оформлении заказа.
- **Конструктор:** `constructor(container: HTMLElement, events: IEvents)` — слушает клик на кнопку "За новыми покупками!" (`success:close`).
- **Поля:** `descriptionElement: HTMLElement`, `closeButtonElement: HTMLButtonElement`.
- **Методы:** `set total(val: number)` — выводит сообщение о количестве списанных синапсов.

### Презентер

Поскольку проект состоит всего из одной страницы, вся логика связи между данными и отображением вынесена в основной скрипт приложения — `main.ts`.

**Как работает Презентер в `main.ts`:**
- Инициализирует API, `EventEmitter`, Модели (`Catalog`, `Cart`, `Buyer`).
- Создаёт экземпляры представлений (`Gallery`, `Header`, `ModalContainer` и др.), передавая им заготовленные шаблоны (через `cloneTemplate`) и брокер событий.
- **Слушает события изменения Моделей** (например, `catalog:updated`, `cart:updated`, `buyer:updated`) и при их срабатывании перерисовывает соответствующие View-компоненты.
- **Слушает действия пользователя из View** (например, `cart:toggle`, `order:submit`, `modal:close`) и в ответ вызывает нужные методы Моделей или переключает отображение в `ModalContainer`.
- Осуществляет финальную сборку данных из `Cart` и `Buyer` и отправку сформированного заказа на сервер через `Api`, обрабатывая успешный результат открытием компонента `SuccessModal`.
